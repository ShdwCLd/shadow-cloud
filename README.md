# Shadow Cloud ($SHDW)

**Deploy websites to Ethereum. Impossible to quietly erase.**

Shadow Cloud turns websites, docs, launch pages, public statements, and digital knowledge into permanent Ethereum-owned artifacts. It is not normal hosting, not a storage clone, and not another "run nodes for rewards" wrapper.

It is an internet memory layer.

## Lore

The internet learned to forget.

Links rot. Launch pages vanish. Docs disappear after teams pivot. Roadmaps get rewritten after markets move. Shadow Cloud begins with one rule:

**If a community believes something matters, it can give that thing a body.**

A Shadow object carries proof, ownership, fragments, witnesses, and a public survival score. Not every page deserves forever. The market decides what becomes part of permanent internet memory.

## What Shadow Cloud Provides

- **Permanent Launch Pages** for ETH projects, memecoins, DAOs, and launches.
- **Forever Docs** for whitepapers, roadmaps, specs, and research.
- **Truth Vaults** for public statements that should not be quietly edited.
- **Knowledge Capsules** for books, articles, alpha, education, and archives.
- **Personal Pages** for wallet-native portfolios, resumes, and builder identity.

## How It Works

```txt
upload site
  -> encrypt
  -> fragment
  -> store
  -> mint ownership/proof on Ethereum
  -> reconstruct locally when opened
```

Current frontend shows this as an interactive product prototype. Backend, wallet auth, contracts, and storage network come next.

## User Flow

1. Connect wallet.
2. Upload website, docs, or vault content.
3. Choose permanence level.
4. Mint Shadow object.
5. Share `shadow://...` link or normal web gateway.
6. Community backs important artifacts with $SHDW.

## Earning Model

Shadow Cloud is designed around **Shadow Guardians**.

Guardians preserve fragments for categories they care about:

- crypto history
- science
- journalism
- open-source
- memes
- education

Planned earning loops:

- Store fragments.
- Prove uptime.
- Serve reconstruction requests.
- Earn $SHDW rewards.
- Gain public Guardian rank.

Important: rewards, token mechanics, and Guardian network are planned protocol features. They are not live in this frontend prototype yet.

## $SHDW Utility

Planned utility:

## Shadow Score

Each artifact gets a survival score based on:

- replicas
- uptime
- community backing
- citations
- access frequency

Higher score means stronger resilience.

## Viral Feature

**Attempt Removal**

Every Shadow page can show a takedown simulation:

```txt
Attempting takedown...
Failed.
Reconstructed from shadow fragments.
```

The goal is simple: make permanence visible.
